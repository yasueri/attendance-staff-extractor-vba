====================================================================
シフト表 CSV出力ツール（VBA） README
====================================================================

■ 概要
--------------------------------------------------------------------
「VBA実行シート」のボタンから実行することで、月間シフト表（例：「6月」
シート）から出勤者・役割・勤務時間・残業有無を1か月分まとめて抽出し、
1つのCSVファイルとして書き出すツールです。

出力されたCSVは、別途作成するJavaScriptプログラムに読み込ませ、
指定した日にちの勤務者情報を抽出・配置する用途を想定しています。


■ ファイル構成
--------------------------------------------------------------------
・ExportShiftCSV.txt
    VBAのコード本体。VBAエディタの標準モジュールに貼り付けて使用する。

・parseShiftCSV_reference.js
    出力したCSVをJavaScript側で読み込むための参考コード（RFC4180準拠の
    CSVパーサー）。外部ライブラリ不要でそのまま利用可能。

・readme.txt
    このファイル。


■ セットアップ手順
--------------------------------------------------------------------
1. Excelファイルを開き、Alt + F11 でVBAエディタを起動する。

2. メニューから「挿入」→「標準モジュール」を選び、新しいモジュールを
   作成する。

3. ExportShiftCSV.txt の中身を全選択してコピーし、作成した標準モジュール
   に貼り付ける。

4. 「VBA実行シート」に配置してあるボタンを右クリック →
   「マクロの登録」→ 全日の出勤者抽出_ を選択する。

5. 「VBA実行シート」C14セルに、シフト表で残業を示す塗りつぶし色を設定
   する（下記「設定項目一覧」参照）。


■ 「VBA実行シート」の設定項目一覧
--------------------------------------------------------------------
ラベル（B列）は説明用の文字列で、実際にVBAが読み取るのはC列の値です。

セル   内容                                        入力例
----   ------------------------------------------  --------
C5     西暦何年のシフト？                           2026
C6     何月のシフト？（1～12）                      6
C7     シフトが入力されているシートの名前は？        6月
C9     月初の日付（1日）が入力されている列は？       D
C10    月初の日付（1日）が入力されている行は？       4
C11    スタッフ名が入力されている列は？              B
C12    最初のスタッフは何行目？                      6
C13    スタッフ検索は何行目まで行う？                30
C14    残業セルの色は？（セルの背景色そのものを設定）

【C14の設定方法（重要）】
パレットから似た色を手動で選び直すと、RGB値が微妙にズレて一致判定に
失敗することがあります。シフト表の実際の残業セルを1つ選択し、
「書式のコピー/貼り付け」（コピー元セルを選択→Ctrl+C→C14上で右クリック
→形式を選択して貼り付け→書式）でC14に色だけを転写する方法を推奨します。

【C9（日付開始列）について】
以前のバージョンでは「スタッフ名の列の2つ右が1日目」という前提で自動計算
していましたが、シフト表の作りが変わっても対応できるよう、日付が始まる
列を独立して指定できるようにしました。スタッフ名の列とは無関係に、実際に
1日目の日付が入っているセルの列を指定してください。


■ 「スタッフ一覧」シートの構成
--------------------------------------------------------------------
A列＝氏名、B列＝役割という構成を前提にしています。

シート上部に何行タイトルや説明文があってもかまいません。VBAはA列を
上から探索し、「氏名」という文字列が入っているセル（見出し行）を自動的
に見つけ、その次の行からデータとして読み込みます。そのため、見出しが
1行目にあっても、5行目にあっても、後で行数が増減しても、コード側の
変更は一切不要です。

例：
  A1: タイトル：スタッフ一覧
  A2: 説明：〇〇
  A3: （空欄）
  A4: 氏名          B4: 役割   ← この行が自動検出される
  A5: 山田　一郎     B5: 店長
  A6: 佐藤　二郎     B6: 主任
  A7: 後藤　四郎     B7: （空欄）

役割が未入力のスタッフがいても問題ありません（空欄のまま出力されます）。


■ シフト表側の前提条件
--------------------------------------------------------------------
・氏名は2行結合されており、1行目が「開始」の行、2行目が「終了」の行。
・開始・終了の時刻は、数値（時刻シリアル値、セル書式が「[h]:mm」等）と
  文字列（「7:30」のようなテキスト入力）のどちらでも正しく処理されます。
  数値の場合は24時間を超える経過時間（例：翌日8:00を意味する「32:00」）
  にも対応しています。
・休みの日は、開始または終了のセルが空欄、あるいは「休」等コロンを
  含まない文字列になっていること（「有給」「希望休」等、表記が何で
  あってもコロンを含まなければ休みとして扱われます）。
・残業（前残業／後残業）は、該当する開始／終了セルの塗りつぶし色が
  C14セルの色と完全一致するかどうかで判定します（条件付き書式では
  なく、セル自体に直接色を設定している運用を前提としています）。


■ 出力されるCSVの仕様
--------------------------------------------------------------------
文字コード：UTF-8（BOM付き）
区切り文字：カンマ
1行目：ヘッダー行

列構成：
  1列目  日付       yyyy-mm-dd 形式（例：2026-06-01）
  2列目  氏名       改行・カンマを含む場合はダブルクォートで囲んで出力
  3列目  開始時刻   シフト表の値をそのまま出力（例：7:30）
  4列目  終了時刻   シフト表の値をそのまま出力（例：22:00、32:00 等）
  5列目  前残業有無 TRUE / FALSE
  6列目  後残業有無 TRUE / FALSE
  7列目  役割       「スタッフ一覧」B列の値（未入力なら空欄）

並び順：① 日付順 → ② シフト表に登場する順（上から下）

「スタッフ一覧」シートに名前が無いスタッフは、シフト表に記載が
あっても出力対象から除外されます。
氏名の照合時は、全角スペース・半角スペース・タブの表記ゆれを
吸収して同一人物として認識します（出力される氏名自体は、シフト表に
入力されている表記のまま）。

【セキュリティ対策】氏名・役割・開始時刻・終了時刻の値が「=」「+」「-」
「@」のいずれかで始まる場合、そのCSVを後でExcelで直接開いた際に数式と
して誤解釈されないよう、先頭にシングルクォート（'）を自動的に付与し
ます（CSVインジェクション対策）。通常の氏名・役割・時刻データでは発生
しません。JavaScript側でこの値を表示する際は、先頭が「'」で、かつ次の
文字が「=+-@」のいずれかの場合のみそれを取り除いてください（同梱の
parseShiftCSV_reference.js は対応済みです）。


■ 実行時のメッセージ一覧
--------------------------------------------------------------------
・「C5セル（西暦何年のシフト？）に数値を入力してください。」
    → C5が空欄、または数値以外が入力されている。

・「C5セル（西暦何年のシフト？：◯◯）が不自然な値です。」
    → 年が1900～2200の範囲外（桁抜け等の入力ミスの可能性）。

・「C6セル（何月のシフト？）は1～12の範囲で入力してください。」
    → C6の値が1～12の範囲外。

・「C7セル（シフトが入力されているシートの名前は？）を入力してください。」
    → C7が空欄。

・「C9セル（月初の日付（1日）が入力されている列は？）には…」
    → C9に列アルファベット（例：D）以外が入力されている。

・「C10／C12セルは1以上の行番号を入力してください。」
    → 行番号が0以下、または数値以外。

・「C11セル（スタッフ名が入力されている列は？）には…」
    → C11に列アルファベット（例：B）以外が入力されている。

・「C13セル（スタッフ検索の最終行）がC12セルより小さくなっています。」
    → C12とC13の大小関係が逆転している。

・「C14セル（残業セルの色は？）に…塗りつぶし色を設定してください。」
    → C14セルに色が設定されていない。

・「シフト表シート「◯◯」が見つかりません。」
    → C7で指定したシート名が実際には存在しない。

・「スタッフ一覧」シートに氏名が見つかりません。」
    → 「スタッフ一覧」シートのA列に「氏名」という見出し、または見出し
      以降のデータが見つからない。

・「シフト表の◯◯行目に日付データが見つかりません。」
    → C10で指定した行に日付らしき数値データが1つも見つからない。

・「シフト表の◯◯行目に、日付が入っていないセルがあります。」
    → 日付行の途中に空欄がある（コピペミス等の可能性）。具体的な
      セル位置が表示されるので、そこを確認する。

・「シフト表の日付（◯◯日）が、C6セルで指定した月（◯月）の範囲を
   超えています。」
    → シフト表内の日付データが、C6で指定した月と矛盾している
      （前月シートを流用した際のコピペミス等が主な原因）。

・完了時：「CSVファイルを出力しました。（保存先パス） 出力件数：◯件」
    → 正常終了。出力件数が想定と大きく異なる場合は設定を再確認。


■ JavaScript側で読み込む際の注意点
--------------------------------------------------------------------
氏名フィールドに改行が含まれるケース（例：「佐藤(改行)（サポート）」）
があるため、単純に文字列をカンマや改行で分割するプログラムでは正しく
読み込めません。ダブルクォートで囲まれたフィールド内の改行・カンマ・
エスケープされた二重引用符を正しく扱えるCSVパーサーを使用してください。

同梱の parseShiftCSV_reference.js は、この仕様に対応した参考実装です。
外部ライブラリ・インターネット接続は不要で、そのままブラウザで動作
します。新しいチャットでJavaScript側を作成する際は、この仕様（UTF-8
BOM付き、RFC4180準拠、列構成、並び順、CSVインジェクション対策）を伝え
た上で、このファイルを渡して「これをベースに実装して」と指示すると
実装ミスを防げます。


■ 既知の制約・前提（今後シート構成が変わった場合に確認する点）
--------------------------------------------------------------------
・時刻セルは数値（時刻シリアル値）・文字列のどちらの入力にも対応済み。
・残業判定はセル自体の塗りつぶし色（直接指定）による判定。条件付き
  書式を使う運用に変わった場合はロジックの変更が必要。
・「スタッフ一覧」はA列に「氏名」という見出しがあり、B列がその見出しと
  同じ行に「役割」、以降の行に氏名・役割が並ぶ前提。列の位置（A＝氏名、
  B＝役割）自体が変わる場合はコードの修正が必要（行数の増減は自動対応
  済み）。
・シフト表は氏名2行結合、1行目が開始・2行目が終了という構成が前提。
・1か月あたりのスタッフ数・日数は少人数～数十人規模を想定した作りで、
  数百人規模になる場合はセルアクセス方式の見直し（配列一括読み込み等）
  を検討した方がよい。
