====================================================================
シフト表 CSV出力ツール
====================================================================
※日跨ぎの場合は24時間を超えた時間表記版
　（例：夜勤で翌日の朝8:00まで勤務　→　32:00

Excelのシフト表から、日々の出勤者・勤務時間・残業有無・役割をまとめて
抽出し、CSVファイルとして出力するExcel VBAマクロです。出力したCSVは、
別途用意するJavaScriptプログラム等で読み込み、日付ごとの勤務者情報を
表示・配置する用途を想定しています。

■ 1. ファイル構成
--------------------------------------------------------------------
ExportShiftCSV.txt
    VBAのコード本体。VBAエディタの標準モジュールに貼り付けて使用する。

parseShiftCSV_reference.js
    出力したCSVをJavaScript側で読み込むための参考実装（CSVパーサー）。
    外部ライブラリ不要、ブラウザでそのまま動作する。

readme.txt
    このファイル。


■ 2. 動作環境
--------------------------------------------------------------------
・Windows + Excel（VBA、ADODB.Stream、Scripting.Dictionary を使用。
  いずれもWindows標準のコンポーネントで、追加のインストールは不要）
・インターネット接続不要（完全にオフラインで動作する）


■ 3. Excelブックの構成
--------------------------------------------------------------------
このマクロは、次の3シートを含むExcelブックを前提にしています。

・VBA実行シート
    マクロの実行ボタンと各種設定値を配置するシート。
・シフト表シート（シート名は任意。「VBA実行シート」C5セルで指定）
    月間の出勤予定を管理するシート。
・スタッフ一覧
    抽出対象のスタッフ氏名・役割を管理するシート。


■ 4. セットアップ手順
--------------------------------------------------------------------
1. Excelファイルを開き、Alt + F11 でVBAエディタを起動する。

2. メニューから「挿入」→「標準モジュール」を選び、新しいモジュールを
   作成する。

3. ExportShiftCSV.txt の中身を全選択してコピーし、作成した標準モジュール
   に貼り付ける。

4. 「VBA実行シート」に配置したボタンを右クリック →
   「マクロの登録」→ 全日の出勤者抽出_ を選択する。

5. 「VBA実行シート」C12セルに、シフト表で残業を示す塗りつぶし色を設定
   する（詳細は次項「設定項目一覧」を参照）。


■ 5. 「VBA実行シート」の設定項目一覧
--------------------------------------------------------------------
ラベルはB列に説明用の文字列として入力し、実際にマクロが読み取るのは
C列の値です。

セル   内容                                        入力例
----   ------------------------------------------  --------
C5     シフトが入力されているシートの名前は？        6月
C7     月初の日付（1日）が入力されている列は？       D
C8     月初の日付（1日）が入力されている行は？       4
C9     スタッフ名が入力されている列は？              B
C10    最初のスタッフは何行目？                      6
C11    スタッフ検索は何行目まで行う？                30
C12    残業セルの色は？（セルの背景色そのものを設定）

【年・月について】
シフト表の日付セルには完全な日付（例：2026/6/1。表示形式が「d」で
「1」としか見えなくても、セルの実体は日付データ）が入力されている前提
のため、年・月を別途指定する設定項目はありません。1日目の日付から
マクロが自動的に年月を取得し、以降の日付がその年月と一致するかどうか
の確認にも使用します。

【C12（残業色）の設定方法】
カラーパレットから似た色を目視で選び直すと、RGB値が微妙にズレて一致
判定に失敗することがあります。シフト表の実際の残業セルを1つ選択し、
「書式のコピー/貼り付け」（コピー元セルを選択→Ctrl+C→C12セル上で
右クリック→形式を選択して貼り付け→書式）でC12に色だけを転写する方法
を推奨します。

【C7（日付開始列）について】
スタッフ名の列とは無関係に、実際に1日目の日付が入っているセルの列を
指定してください。


■ 6. 「スタッフ一覧」シートの構成
--------------------------------------------------------------------
A列＝氏名、B列＝役割という構成を前提にしています。

  A1: タイトル：スタッフ一覧
  A2: 説明：〇〇
  A3: （空欄）
  A4: 氏名          B4: 役割   ← この行が自動検出される
  A5: 山田　一郎     B5: 店長
  A6: 佐藤　二郎     B6: 主任
  A7: 後藤　四郎     B7: （空欄）

シート上部に何行タイトルや説明文があってもかまいません。マクロはA列を
上から探索し、「氏名」という文字列が入っているセル（見出し行）を自動的
に見つけ、その次の行からデータとして読み込みます。そのため、見出しが
1行目にあっても5行目にあっても、後で行数が増減しても、コード側の変更
は不要です。役割が未入力のスタッフがいても問題ありません（空欄のまま
出力されます）。

なお、「スタッフ一覧」に名前が無いスタッフは、シフト表に記載があって
も出力対象から除外されます。氏名の照合時は、全角スペース・半角スペー
ス・タブの表記ゆれを吸収して同一人物として認識します（出力される氏名
自体は、シフト表に入力されている表記のまま出力されます）。


■ 7. シフト表シートの構成・前提条件
--------------------------------------------------------------------
・氏名は2行結合されており、1行目が「開始」の行、2行目が「終了」の行。
  （例：氏名がB6:B7に結合、B6が開始行、B7が終了行）
・日付行には完全な日付が入力されていること。表示形式は「d」のように
  日にちだけを表示するものでもかまわない（マクロはセルの実体の値を
  読み取るため、表示形式には依存しない）。
・開始・終了の時刻は、次のどちらの入力方法にも対応している。
    - 数値（時刻シリアル値。セル書式が「[h]:mm」等）
    - 文字列（「7:30」のようなテキスト入力）
  数値の場合、24時間を超える経過時間（例：翌日8:00を意味する「32:00」）
  にも対応している。
・休みの日は、開始または終了のセルが空欄、あるいは「休」等コロンを
  含まない文字列になっていること。「有給」「希望休」等、表記が何で
  あってもコロンを含まなければ休みとして扱われる。
・残業（前残業／後残業）は、該当する開始／終了セルの塗りつぶし色が
  C12セルの色と完全一致するかどうかで判定する（条件付き書式ではなく、
  セル自体に直接色を設定している運用が前提）。


■ 8. 出力されるCSVの仕様
--------------------------------------------------------------------
文字コード：UTF-8（BOM付き）
区切り文字：カンマ
1行目：ヘッダー行

列構成：
  1列目  日付       yyyy-mm-dd 形式（例：2026-06-01）
  2列目  氏名       改行・カンマを含む場合はダブルクォートで囲んで出力
  3列目  開始時刻   H:MM形式（例：7:30、22:00、32:00）
  4列目  終了時刻   H:MM形式
  5列目  前残業有無 TRUE / FALSE
  6列目  後残業有無 TRUE / FALSE
  7列目  役割       「スタッフ一覧」B列の値（未入力なら空欄）

並び順：① 日付順 → ② シフト表に登場する順（上から下）

出力例：
  日付,氏名,開始時刻,終了時刻,前残業,後残業,役割
  2026-06-01,山田　一郎,7:30,16:30,FALSE,FALSE,店長
  2026-06-01,後藤　四郎,11:00,22:00,TRUE,FALSE,

【CSVインジェクション対策】
氏名・役割・開始時刻・終了時刻の値が「=」「+」「-」「@」のいずれかで
始まる場合、そのCSVを後でExcelで直接開いた際に数式として誤解釈されな
いよう、先頭にシングルクォート（'）を自動的に付与する。通常の氏名・
役割・時刻データでは発生しない。JavaScript側でこの値を表示する際は、
先頭が「'」で、かつ次の文字が「=+-@」のいずれかの場合のみそれを取り
除く（同梱の parseShiftCSV_reference.js は対応済み）。


■ 9. 実行時のメッセージ一覧
--------------------------------------------------------------------
・「C5セル（シフトが入力されているシートの名前は？）を入力してください。」
    → C5が空欄。

・「C7セル（月初の日付（1日）が入力されている列は？）には…」
    → C7に列アルファベット（例：D）以外が入力されている。

・「C8／C10セルは1以上の行番号を入力してください。」
    → 行番号が0以下、または数値以外。

・「C9セル（スタッフ名が入力されている列は？）には…」
    → C9に列アルファベット（例：B）以外が入力されている。

・「C11セル（スタッフ検索の最終行）がC10セルより小さくなっています。」
    → C10とC11の大小関係が逆転している。

・「C12セル（残業セルの色は？）に…塗りつぶし色を設定してください。」
    → C12セルに色が設定されていない。

・「シフト表シート「◯◯」が見つかりません。」
    → C5で指定したシート名が実際には存在しない。

・「スタッフ一覧」シートに氏名が見つかりません。」
    → 「スタッフ一覧」シートのA列に「氏名」という見出し、または見出し
      以降のデータが見つからない。

・「シフト表の◯◯行目に日付データが見つかりません。」
    → C8で指定した行に日付データが1つも見つからない。

・「シフト表の◯◯行目に、日付が入っていないセルがあります。」
    → 日付行の途中に空欄がある（コピペミス等の可能性）。具体的な
      セル位置が表示されるので、そこを確認する。

・「シフト表の日付（◯◯）が、1日目の日付（◯年◯月）と異なる年月に
   なっています。」
    → シフト表内の日付データが、1日目の日付と矛盾する年月になって
      いる（前月・翌月のシートを流用した際のコピペミス等が主な原因）。

・完了時：「CSVファイルを出力しました。（保存先パス） 出力件数：◯件」
    → 正常終了。出力件数が想定と大きく異なる場合は設定を再確認。

・「エラーが発生しました：（内容）」
    → 上記以外の想定外のエラー。Excelのエラー内容がそのまま表示される。


■ 10. JavaScript側で読み込む際の注意点
--------------------------------------------------------------------
氏名フィールドに改行が含まれるケース（スタッフ一覧の氏名に改行を含む
表記がある場合など）があるため、単純に文字列をカンマや改行で分割する
プログラムでは正しく読み込めない。ダブルクォートで囲まれたフィールド
内の改行・カンマ・エスケープされた二重引用符を正しく扱えるCSVパーサー
を使用すること。

同梱の parseShiftCSV_reference.js は、この仕様に対応した参考実装。
外部ライブラリ・インターネット接続は不要で、そのままブラウザで動作
する。CSVを扱うプログラムを別途作成する場合は、この仕様（UTF-8 BOM
付き、RFC4180準拠、列構成、並び順、CSVインジェクション対策）を伝えた
上で、このファイルをベースに実装するとミスを防げる。


■ 11. 既知の制約
--------------------------------------------------------------------
・日付行は完全な日付（表示形式は問わない）が入力されている前提。
・時刻セルは数値（時刻シリアル値）・文字列のどちらの入力にも対応済み。
・残業判定はセル自体の塗りつぶし色（直接指定）による判定。条件付き
  書式を使う運用の場合はロジックの変更が必要。
・「スタッフ一覧」はA列に「氏名」という見出しがあり、B列がその見出しと
  同じ行に「役割」、以降の行に氏名・役割が並ぶ前提。列の位置（A＝氏名、
  B＝役割）自体が変わる場合はコードの修正が必要（行数の増減は自動対応
  済み）。
・シフト表は氏名2行結合、1行目が開始・2行目が終了という構成が前提。
・1か月あたりのスタッフ数・日数は少人数～数十人規模を想定しており、
  数百人規模になる場合はセルアクセス方式の見直し（配列一括読み込み等）
  を検討した方がよい。
