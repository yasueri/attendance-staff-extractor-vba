シフト表 CSV出力ツール（VBA） README
====================================================================

■ 概要
--------------------------------------------------------------------
「VBA実行シート」のボタンから実行することで、月間シフト表（例：「6月」
シート）から出勤者・勤務時間・残業有無を1か月分まとめて抽出し、
1つのCSVファイルとして書き出すツールです。

出力されたCSVは、別途作成するJavaScriptプログラムに読み込ませ、
指定した日にちの勤務者情報を抽出・配置する用途を想定しています。


■ ファイル構成
--------------------------------------------------------------------
・ExportShiftCSV.txt
    VBAのコード本体。VBAエディタの標準モジュールに貼り付けて使用する。

・parseShiftCSV_reference.js
    出力したCSVをJavaScript側で読み込むための参考コード（RFC4180準拠の
    CSVパーサー）。外部ライブラリ不要でそのまま利用可能。

・readme.txt
    このファイル。


■ セットアップ手順
--------------------------------------------------------------------
1. Excelファイルを開き、Alt + F11 でVBAエディタを起動する。

2. メニューから「挿入」→「標準モジュール」を選び、新しいモジュールを
   作成する。

3. ExportShiftCSV.txt の中身を全選択してコピーし、作成した標準モジュール
   に貼り付ける。

4. 「VBA実行シート」に配置してあるボタンを右クリック →
   「マクロの登録」→ ExportShiftCSV を選択する。

5. 「VBA実行シート」B13セルに、シフト表で残業を示す塗りつぶし色を設定
   する（下記「設定項目一覧」参照）。


■ 「VBA実行シート」の設定項目一覧
--------------------------------------------------------------------
セル   内容                                        入力例
----   ------------------------------------------  --------
B5     シフトの年                                   2026
B6     シフトの月（1～12）                          6
B7     シフト表が入力されているシート名               6月
B9     シフト表でスタッフ名が入力されている列         B
B10    シフト表で日付が入力されている行               4
B11    シフト表で最初のスタッフが出てくる行           6
B12    シフト表でスタッフを検索する最終行             30
B13    残業時の塗りつぶし色（セルの背景色そのものを設定）

【B13の設定方法（重要）】
パレットから似た色を手動で選び直すと、RGB値が微妙にズレて一致判定に
失敗することがあります。シフト表の実際の残業セルを1つ選択し、
「書式のコピー/貼り付け」（コピー元セルを選択→Ctrl+C→B13上で右クリック
→形式を選択して貼り付け→書式）でB13に色だけを転写する方法を推奨します。


■ シフト表側の前提条件
--------------------------------------------------------------------
・氏名は2行結合されており、1行目が「開始」の行、2行目が「終了」の行。
・開始・終了の時刻は「7:30」のようにコロンを含む文字列（テキスト形式）
  として入力されていること。実際のExcelの時刻シリアル値が入っている
  場合は、CSVEscape に渡す前に Format(value, "h:mm") 等での変換が
  別途必要（現状のコードはテキスト入力を前提としています）。
・休みの日は、開始または終了のセルが空欄、あるいは「休」等コロンを
  含まない文字列になっていること（「有給」「希望休」等、表記が何で
  あってもコロンを含まなければ休みとして扱われます）。
・残業（前残業／後残業）は、該当する開始／終了セルの塗りつぶし色が
  B13セルの色と完全一致するかどうかで判定します（条件付き書式では
  なく、セル自体に直接色を設定している運用を前提としています）。


■ 出力されるCSVの仕様
--------------------------------------------------------------------
文字コード：UTF-8（BOM付き）
区切り文字：カンマ
1行目：ヘッダー行

列構成：
  1列目  日付       yyyy-mm-dd 形式（例：2026-06-01）
  2列目  氏名       改行・カンマを含む場合はダブルクォートで囲んで出力
  3列目  開始時刻   シフト表の値をそのまま出力（例：7:30）
  4列目  終了時刻   シフト表の値をそのまま出力（例：22:00、32:00 等）
  5列目  前残業有無 TRUE / FALSE
  6列目  後残業有無 TRUE / FALSE

並び順：① 日付順 → ② シフト表に登場する順（上から下）

「スタッフ一覧」シートに名前が無いスタッフは、シフト表に記載が
あっても出力対象から除外されます。
氏名の照合時は、全角スペース・半角スペース・タブの表記ゆれを
吸収して同一人物として認識します（出力される氏名自体は、シフト表に
入力されている表記のまま）。


■ 実行時のメッセージ一覧
--------------------------------------------------------------------
・「B5セル（シフトの年）に数値を入力してください。」
    → B5が空欄、または数値以外が入力されている。

・「B5セル（シフトの年：◯◯）が不自然な値です。」
    → 年が1900～2200の範囲外（桁抜け等の入力ミスの可能性）。

・「B6セル（シフトの月）は1～12の範囲で入力してください。」
    → B6の値が1～12の範囲外。

・「B7セル（シフト表のシート名）を入力してください。」
    → B7が空欄。

・「B9セル（スタッフ名が入力されている列）には…」
    → B9に列アルファベット（例：B）以外が入力されている。

・「B10／B11セルは1以上の行番号を入力してください。」
    → 行番号が0以下、または数値以外。

・「B12セル（スタッフ検索の最終行）がB11セルより小さくなっています。」
    → B11とB12の大小関係が逆転している。

・「B13セル（残業時のシート色）に…塗りつぶし色を設定してください。」
    → B13セルに色が設定されていない。

・「シフト表シート「◯◯」が見つかりません。」
    → B7で指定したシート名が実際には存在しない。

・「スタッフ一覧」シートに氏名が見つかりません。」
    → 「スタッフ一覧」シートに「氏名」という見出し、または見出し以降の
      データが見つからない。

・「シフト表の◯◯行目に日付データが見つかりません。」
    → B10で指定した行に日付らしき数値データが1つも見つからない。

・「シフト表の日付（◯◯日）が、B6セルで指定した月（◯月）の範囲を
   超えています。」
    → シフト表内の日付データが、B6で指定した月と矛盾している
      （前月シートを流用した際のコピペミス等が主な原因）。

・完了時：「CSVファイルを出力しました。（保存先パス） 出力件数：◯件」
    → 正常終了。出力件数が想定と大きく異なる場合は設定を再確認。


■ JavaScript側で読み込む際の注意点
--------------------------------------------------------------------
氏名フィールドに改行が含まれるケース（例：「佐藤(改行)（サポート）」）
があるため、単純に文字列をカンマや改行で分割するプログラムでは正しく
読み込めません。ダブルクォートで囲まれたフィールド内の改行・カンマ・
エスケープされた二重引用符を正しく扱えるCSVパーサーを使用してください。

同梱の parseShiftCSV_reference.js は、この仕様に対応した参考実装です。
外部ライブラリ・インターネット接続は不要で、そのままブラウザで動作
します。新しいチャットでJavaScript側を作成する際は、この仕様（UTF-8
BOM付き、RFC4180準拠、列構成、並び順）を伝えた上で、このファイルを
渡して「これをベースに実装して」と指示すると実装ミスを防げます。


■ 既知の制約・前提（今後シート構成が変わった場合に確認する点）
--------------------------------------------------------------------
・時刻セルはテキスト入力（コロンを含む文字列）である前提。
・残業判定はセル自体の塗りつぶし色（直接指定）による判定。条件付き
  書式を使う運用に変わった場合はロジックの変更が必要。
・「スタッフ一覧」はA列に「氏名」という見出しがあり、その下に氏名が
  並ぶ前提。
・シフト表は氏名2行結合、1行目が開始・2行目が終了という構成が前提。
・1か月あたりのスタッフ数・日数は少人数～数十人規模を想定した作りで、
  数百人規模になる場合はセルアクセス方式の見直し（配列一括読み込み等）
  を検討した方がよい。
